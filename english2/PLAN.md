# 개발 계획

## 1단계: 데이터 파이프라인

- xlsx 단어장을 `data/words.json`으로 변환한다.
- 단어, 뜻, 품사, 예문, 단원, 난이도를 정규화한다.
- 원본 행 정보와 시트명을 남겨 검수 가능하게 한다.
- `scripts/prepare_data.py`로 변환, 검증, 정적 빌드를 한 번에 실행한다.
- `scripts/report_data.py`로 단원별 개수, 누락 필드, 중복 후보 리포트를 생성한다.
- 변환기 회귀 테스트로 열 감지, 무헤더 파일, 중복 ID를 확인한다.

## 2단계: 학습 앱

- 검색 가능한 단어장
- 단원별 필터
- 플래시카드 학습
- 객관식/입력형 퀴즈
- 즐겨찾기와 오답노트
- 브라우저 로컬 저장 기반 진도 저장
- 진도 내보내기/가져오기로 모바일과 PC 간 백업 이동
- PWA 설치와 오프라인 캐시 지원
- 일일 학습 목표와 날짜별 학습량 기록
- 단어장 상태 필터로 미학습, 복습, 오답, 즐겨찾기, 완료 항목 검수

## 3단계: 배포

- GitHub Pages용 정적 앱으로 배포한다.
- 원본 xlsx는 저장소에 넣지 않고, 변환된 JSON만 앱에 포함한다.
- 배포 전에 `scripts/validate_data.py`와 `scripts/build_pages.py`를 실행한다.
- `scripts/check_all.py`로 JS 문법, Python 문법, 변환기 테스트, 데이터 검증, 정적 빌드를 한 번에 실행한다.
- `scripts/package_upload.py`로 Git 명령 없이 업로드 가능한 ZIP을 생성한다.
- 공개 URL 노출 위험이 있으면 인증 가능한 배포 방식으로 전환한다.

## 4단계: AI 기능

- 정적 GitHub Pages에서는 API 키를 안전하게 숨길 수 없으므로 서버리스 백엔드가 필요하다.
- 이후 후보:
  - 예문 생성
  - 단어별 뉘앙스 비교
  - 오답 기반 퀴즈 생성
  - 학습 리포트 요약
